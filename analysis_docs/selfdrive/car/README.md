# selfdrive/car 概要

このディレクトリはopenpilotの「車両ごとの制御・適合」ロジックをまとめた中核部分です。車種ごとの挙動差分や、車両制御の共通処理、テストなどが集約されています。

## 目的・背景
- openpilotは多種多様な車種に対応するため、車両ごとの制御ロジックやパラメータを柔軟に切り替える必要がある
- このディレクトリでは、車種ごとの個別実装・共通処理・テスト・ドキュメントを整理
- 新しい車種を追加する際も、この構造に従って拡張できる

## 主な構成
- `car_specific.py` : 車種ごとの個別制御ロジックやパラメータを定義
- `card.py` : 車両情報カードやUI表示用のデータ管理
- `cruise.py` : クルーズコントロール（ACC等）の共通処理
- `docs.py` : 車両ごとの仕様や解説をまとめたドキュメント生成用スクリプト
- `tests/` : 車両制御ロジックの自動テスト
- `CARS_template.md` : 新規車種追加時のテンプレート

## 使い方の例
- 新しい車種を追加したい場合は`CARS_template.md`を参考に`car_specific.py`等を編集
- 車両制御の共通処理を変更したい場合は`cruise.py`を修正
- テストは`tests/`配下で実行

## 注意点
- 車両ごとの挙動差分は安全性に直結するため、十分なテストとレビューが必須
- ドキュメントやテンプレートも活用し、他の開発者が理解しやすい構造を保つ
